목차

  1. 유저 목록 조회 및 검색

  4. 백업 생성 (수동)
  5. 특정 시점 롤백
  6. 비밀번호 재설정
  7. 계정 사용 중지 / 활성화
  8. 계정 삭제
  9. 작업 흐름 예시

1. 유저 목록 조회 및 검색

1 유저 관리 탭으로 이동
2 "유저 목록 불러오기" 버튼 클릭
3 전체 유저가 테이블로 표시됨 (이름, 이메일, 레벨, 상태, UID)
4 상단 검색창에 이름 / 이메일 / UID 입력 → 실시간 필터링
5 유저 행을 클릭하면 상세 패널이 열림

2. 특정 유저 데이터 초기화

유저의 게임 데이터(레벨, 스탯, 포인트, 퀘스트 등)를 초기 상태로 리셋합니다.
이름, 푸시 설정, FCM 토큰은 유지됩니다.

1 유저 목록에서 대상 유저 클릭
2 상세 패널 → 데이터 관리 섹션
3 "데이터 초기화" 빨간 버튼 클릭
4 확인 다이얼로그에서 승인

초기화 후 상태

level1
points0
stats / pendingStats전부 0
friends빈 배열
퀘스트 / 다이어리 / 릴스 등삭제됨

* 초기화 전 자동 백업이 생성되므로 실수 시 롤백 가능

3. 전체 유저 데이터 초기화

모든 유저의 게임 데이터를 한 번에 초기화합니다.

1 유저 관리 탭 하단 "전체 유저 데이터 초기화" 카드
2 "전체 초기화" 빨간 버튼 클릭
3 2단계 확인 다이얼로그 통과 (실수 방지)
4 각 유저마다 자동 백업 생성 후 초기화 진행
5 완료 시 초기화된 유저 수가 알림으로 표시됨

4. 백업 생성 (수동)

데이터 변경 전 수동으로 스냅샷을 저장합니다.

1 유저 선택 → 상세 패널
2 "백업 생성" 클릭
3 메모 입력 (예: "이벤트 전 백업", "버그 수정 전") → 확인

자동 백업이 생성되는 경우

메모생성 시점
수동 백업관리자가 직접 생성
데이터 초기화 전 자동 백업개별 유저 초기화 시
전체 초기화 전 자동 백업전체 초기화 시
롤백 전 자동 백업롤백 실행 시
계정 삭제 전 자동 백업계정 삭제 시

5. 특정 시점 롤백

백업된 시점의 데이터로 유저를 복원합니다.

1 유저 선택 → 상세 패널
2 "백업 목록" 클릭 → 백업 이력 테이블 표시 (시점, 메모, 생성자)
3 원하는 백업의 "롤백" 버튼 클릭
4 확인 다이얼로그 승인

* 롤백 전 현재 상태도 자동 백업됨 → 롤백의 롤백도 가능

6. 비밀번호 재설정

Firebase Auth 비밀번호 리셋 링크를 생성합니다. (이메일/비밀번호 로그인 유저 대상)

1 유저 선택 → 상세 패널 → 계정 관리 섹션
2 "비밀번호 재설정" 클릭
3 리셋 링크가 입력 필드에 표시됨
4 링크를 클릭 → 복사 후 유저에게 전달
주의: Google 로그인 전용 유저는 이메일이 없을 수 있으며, 이 경우 오류가 표시됩니다.

7. 계정 사용 중지 / 활성화

유저의 로그인을 차단하거나 복원합니다. 데이터는 유지됩니다.

1 유저 선택 → 상세 패널
2 "계정 사용 중지" 클릭 → 확인
3 해당 유저는 더 이상 로그인 불가 (앱에서 자동 로그아웃)
4 유저 목록에서 상태가 중지됨 으로 변경
5 다시 선택하면 버튼이 "계정 활성화"로 바뀜 → 클릭 시 복원

8. 계정 삭제

Firebase Auth 계정 + Firestore 유저 문서를 영구 삭제합니다.

1 유저 선택 → 상세 패널
2 "계정 삭제" 빨간 버튼 클릭
3 2단계 확인 다이얼로그 통과

삭제 범위

항목삭제 여부
users/{uid} Firestore 문서삭제됨
Firebase Auth 계정삭제됨 (복구 불가)
Cloud Storage 프로필 이미지수동 정리 필요
reels_reactions 좋아요/댓글수동 정리 필요
user_backups 백업 데이터보존됨 (의도적)

* 삭제 전 자동 백업이 생성됩니다 (Firestore 데이터만 복구 가능)

9. 작업 흐름 예시

예시 1 — 버그로 스탯 비정상 → 어제 상태로 복원

유저 관리 탭 → 유저 검색/선택 → 백업 목록 → 어제 날짜 백업 "롤백"

예시 2 — 시즌 리셋 (전체 유저 초기화)

유저 관리 탭 → 하단 "전체 초기화" → 2번 확인 → 완료

* 각 유저 자동 백업이 생성되므로 문제 시 개별 롤백 가능

예시 3 — 악성 유저 처리

유저 검색/선택 → "계정 사용 중지" → 판단 후 "활성화" 또는 "삭제"

예시 4 — 비밀번호 분실 문의

유저 검색/선택 → "비밀번호 재설정" → 링크 복사 → 유저에게 전달
