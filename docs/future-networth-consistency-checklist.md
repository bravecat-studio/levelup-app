# 미래 순자산 카드/상세/칭호 일관성 수동 점검 시나리오

## 목적
`future_networth_config`의 동일 입력(`g`, `r`, `roi`, `e`, `W_0`, `n`)에서 아래 3개 결과가 일관되게 계산되는지 확인합니다.
- 카드의 월 가용 저축력(M_avail)
- 계산 상세 모달의 W_total / M_avail
- 저축 관련 희귀 칭호 기준(rate = M_avail / (W_0/12) × 100)

## 사전 준비
1. 설정 모달에서 아래 샘플 값을 입력하고 저장합니다.
   - `W_0`: 6,000
   - `n`: 10
   - `g`: 4.0
   - `r`: 2.5
   - `e`: 65
   - `inflateS`: OFF
   - 목돈 지출 항목: 전부 0
2. 브라우저 개발자도구 `Application > Local Storage`에서 `future_networth_config`를 열어 `g`, `r`, `roi`, `e` 값이 저장되었는지 확인합니다.

## 점검 절차
1. **카드 값 확인**
   - 미래 순자산 카드에서 `월 가용 저축력(M_avail)` 표시값을 기록합니다.
2. **상세 모달 값 확인**
   - `📊 계산 상세` 버튼을 눌러 `W_total`, `M_avail` 값을 기록합니다.
   - 상단 파라미터 배지(`r`, `g`, `e`)가 Local Storage 값과 일치하는지 확인합니다.
3. **칭호 계산 기준 확인**
   - 아래 식으로 rate를 직접 계산합니다.
     - `rate = M_avail / (W_0 / 12) × 100`
   - 계산된 rate가 희귀 칭호 threshold를 넘는지 확인하고, 앱에서 동일하게 칭호가 해금되는지 확인합니다.
4. **성장률 파라미터 변경 검증 (핵심)**
   - `g`를 `4.0 → 1.0`으로 낮춰 저장합니다.
   - 카드와 상세의 `W_total`, `M_avail`가 모두 동일 방향(감소)으로 바뀌는지 확인합니다.
   - 칭호 기준 rate도 같은 값으로 감소하는지 확인합니다.
   - (중요) `r`만 변경했을 때는(목돈 인플레 OFF 기준) `W_total`/`M_avail`/칭호 rate가 변하지 않아야 합니다.

## 기대 결과
- 카드, 상세 모달, 칭호 기준이 모두 동일한 `M_avail` 기반으로 동작합니다.
- `W_total` 계산은 `r`이 아니라 `g`를 사용합니다.
- 파생 계산에서 사용하는 기본 파라미터는 `future_networth_config`의 `g`, `r`, `roi`, `e`를 기준으로 정규화되어 사용됩니다.
